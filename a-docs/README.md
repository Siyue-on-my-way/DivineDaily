## 🗂️ 文档使用指南

### 新成员入门
1. 先读 `ARCH_AND_DESIGN.md` 了解整体架构
2. 再读 `PROJECT_STATUS.md` 了解当前状态
3. 根据需要查阅具体的设计文档和实施报告

### 开发新功能
1. 查阅 `ARCH_AND_DESIGN.md` 确保符合整体架构
2. 参考 `TECHNICAL_REFERENCE.md` 了解技术实现细节
3. 查看相关的实施报告了解已有功能的实现方式

### 问题排查
1. 查看 `PROJECT_STATUS.md` 了解已知问题
2. 参考 `项目实现状态分析报告.md` 了解历史问题
3. 查阅 `TECHNICAL_REFERENCE.md` 了解技术细节

### 设计评审
1. 对照 `ARCH_AND_DESIGN.md` 检查架构一致性
2. 参考已完成的实施报告了解设计模式
3. 更新 `PROJECT_STATUS.md` 记录新的设计决策

## 📝 文档维护规范

### 文档分类原则
- **design/** - 架构设计、功能设计、系统设计
- **plan/** - 实施计划、完成报告、状态分析
- **assets/** - 数据文件、图片、资源

### 文档命名规范
- 设计文档：`功能名_design.md` 或 `UPPERCASE_NAME.md`
- 实施报告：`功能名实施完成报告.md` 或 `YYYY-MM-DD_description.md`
- 计划文档：`YYYY-MM-DD_plan_name.md`

### 更新频率
- `PROJECT_STATUS.md` - 每次重大功能完成后更新
- `TECHNICAL_REFERENCE.md` - 新增核心技术实现时更新
- 实施报告 - 功能完成后创建，不再修改
- 设计文档 - 设计变更时更新
